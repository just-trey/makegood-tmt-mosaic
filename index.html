v1 STL support is reference mode: the uploaded mesh is shown for alignment/sizing only. Mosaic's own boolean engine only cuts pockets into its own flat parametric plate (disc / rect / round rect) — it does not cut into arbitrary mesh geometry yet. Export produces a plate insert sized to the face above, for you to seat into the real part.
Real multi-part cutting: an assembly is a fixed set of part roles (e.g. a wheel is Top + Cap). Add each role below — Mosaic auto-detects its dominant flat face and cuts pockets directly into that part's own mesh, all sharing one design in real assembled coordinates.
Roles with a library-linked file auto-load when added, if served over a local http server (see README) — otherwise drag-and-drop the file into the row below.

Colors detected

Load an SVG to detect color regions. Each flat fill color becomes its own recess, filled by a separate AMS filament.
Drag one color onto another to merge them into a single filament / AMS slot — or check two+ and use the button. Drop a color onto an existing merged group to add it. Merged groups share one depth; use "unmerge" to split them back.

Export

One STL per color, plus base.stl for the uncut plate. Import all as separate objects in Bambu Studio, group them, then assign a filament per part for AMS mapping.
